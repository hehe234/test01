# 回顾
## 1.为什么使用miniconda?
##### 1.1 因为我们创建不同的项目需要不同的环境，然后我们可以避免在不同的项目中安装环境依赖冲突，所以我们使用miniconda，给每个项目创建独立的环境依赖
### 2.miniconda和我们的pycharm有什么关系？
##### 2.1 pycharm可以简单的理解成我们的支付宝，举例：我们购物需要支付的时候，那么我们的支付宝中需要绑定银行卡，而这个银行卡，咱们就可以理解为我们使用minicoda来创建的每个虚拟的环境。如果我绑定的是农业银行卡，那么当我农业银行中的余额用完来，我可以切换到另一个银行卡
### 3.我们miniconda常用的命令
- 1.查看虚拟环境列表
  - 1.1 conda env list
- 2.创建一个新的虚拟环境
  - 2.1 conda create -n 环境名称 python=版本号 
  - 2.2 上面的过程会停下来等用户输入 y/n, 一般我们都是输入y,下载依赖
- 3.另一种创建一个新虚拟环境的方法，不需要用户输入y
  - 3.1 conda create -n 环境名称 python=版本号 -y
- 4.切换/激活我们指定的虚拟环境
  - 4.1 conda activate 环境名称
- 5.退出当前虚拟环境
    - 5.1 conda deactivate
